Slide 2 text: Noida Institute of Engineering and Technology | 2026 |  |  | 1. Objective of the Project | 
Primary: Develop an intelligent, automated system for competitive Valorant match data.
Automated Data Collection: Web crawler for VLR.gg.

Slide 3 text: Noida Institute of Engineering and Technology | 2026 |  |  | 2. Problem Statement | 
Manual Data Entry: Time-consuming and error-prone.
Lack of Centralized Analytics: VLR.gg is raw data only.
Siloed Information: Hard to query complex scenarios.
Inefficient Scouting: Requires hours of video review.
ScoutAnt Solution: Automates extraction and aggregation.
Slide 4 text: Noida Institute of Engineering and Technology | 2026 |  |  | 3. Abstract | 
ScoutAnt is a Python-based analytical tool for Valorant esports.
Mines data from VLR.gg (maps, scores, players, agents).
Stores data in structured JSON for longitudinal analysis.
Evolved from legacy scripts to a robust, modular crawler.
Empowers teams to focus on strategy rather than data entry.
Slide 5 text: Noida Institute of Engineering and Technology | 2026 |  |  | 4. Existing Systems | 
VLR.gg: Primary source, no API, consumption-only.
Tracker.gg: Focuses on ranked play, not pro tournaments.
Rib.gg: Paid, pre-defined dashboards.
Manual Spreadsheets: High effort, hard to maintain.
ScoutAnt: Custom query flexibility + Professional data depth.
Slide 6 text: Noida Institute of Engineering and Technology | 2026 |  |  | 5. Proposed Methodology | 
Step 1: Reconnaissance - Analyzed VLR.gg URL structure.
Step 2: DOM Analysis - Identified HTML tables and classes.
Step 3: Crawler Architecture - 3-layer loop (Events -> Matches -> Stats).
Step 4: Data Storage - JSON document store.
Slide 7 text: Noida Institute of Engineering and Technology | 2026 |  |  | 6.1 Functional Requirements |

Data Scraping: Extract Map, Teams, Players, Stats.
Data Persistence: Save to match_stats_db.json.
Checkpointing & Duplicate Detection.
